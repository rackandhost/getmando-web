This release improves accessible interaction, error handling, dashboard behavior, and the project's testing and tooling foundations.

## New features

- Added an accessible search engine selector with arrow, Home, End, and Escape navigation.
- Added signal-based notifications, accessible toasts, retries for transient dashboard loading failures, and safer uncaught-error fallback.

## Fixes and improvements

- Kept the header, finder, and categories fixed while the apps area scrolls.
- Aligned toast styling with application cards and added spacing above the app grid.
- Preserved search queries when an external search popup is blocked and replaced development logging with centralized logging.

## Quality and tooling

- Enabled OnPush change detection and migrated synchronous state to Angular signals where appropriate.
- Added automated axe checks, a 70% core coverage gate, Node 22 CI, focused-test detection, linting, and scoped formatting checks.
- Added an architecture guide and documented accessibility testing and linting workflows.
